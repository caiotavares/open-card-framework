CardServices for the SIM cards

`com.gemplus.opencard.service.sim'

This page is intended for SIM cards applications developers; it is dedicated to the Gemplus Sim CardServices Java package(s) (i.e., SIMCardService for GSM 11.11 features and SIMToolkitApplicationService for GSM 11.14 features) that can be used together with the Gemplus CardTerminal and the "OpenCard Framework" to access and use SIM cards and Toolkit applications from Opencard-enabled platforms such as PCs. Among other things, it describes how to download, install, and use this CardService.

Features

The Gemplus SIM CardServices is an implementation of OCF CardServices to access to a GSM card (SIM).

The access to the SIM is specified in the two following GSM standards:

  1. GSM 11.11 (SIM), and

  2. GSM 11.14 (SIM Toolkit).

These CardServices have been developed with the help of Eurecom, a european research institute specialized in communications and networking, and through a student project co-managed by Eurecom and Gemplus. The first alpha-version has been extensively re-designed and re-organized to eventually lead to this 0.1 version. It is however still a work-in-progress that has barely been tested! (see the Warning Note above).

Pointers to the ETSI site are given in the "Related Reference Documentation" section; otherwise, you can find a rough description of the two GSM standards in the student report in the following sections:

  1. SIM Standard (GSM 11.11), and

  2. SIM Application Toolkit Standard (GSM 11.14)

For using Sim CardService

This section shows how to run the simple tests that are provided with the SIM CardServices and how to start writing your own applications.

Two small tests are provided that can be used as samples of main programs using the CardServices:

  • com.gemplus.opencard.service.sim.Test shows how to use BasicSIMFactory to recognize a SIM card and how to use the SIM card features (i.e., related to the GSM 11.11 standard) that are specified in the SIMCardService interface.

    It currently only performs a Status command (but it will hopefully use more SIM features in the future), and

  • com.gemplus.opencard.service.sim.toolkit.Test shows how to use (in the off-card part) the GSM Toolkit features (i.e., related to the GSM 11.14 standard) that are specified in the SIMToolkitApplicationService interface.

    A small simulator of ME (Mobile Equipment) is provided that can only accept two toolkit functions: Display Text and Set Up Menu. Note that a SIM card containing a small toolkit application is necessary in order to use this test.

For using these two tests, proceed as follows:

  1. download and install the distribution.

  2. configure your "opencard.properties" file in order to use the Gemplus BasicSIM CardService factory (i.e., BasicSIMFactory) for the OpenCard.services property and the generic Gemplus CardTerminal factory (i.e., GemplusCardTerminalFactory) for OpenCard.terminals (if you are using a Gemplus reader of course).

    Example: (warning: the lines are not cut!): 

    OpenCard.services = opencard.opt.util.PassThruCardServiceFactory com.gemplus.opencard.service.sim.BasicSIMFactory

OpenCard.terminals = com.gemplus.opencard.terminal.GemplusCardTerminalFactory|mygcr|GemPC410|COM1

  3. test the SIM or the SIM Toolkit package with respectively the sim/Test.java or the sim/toolkit/Test.java programs.

    Example:

    1. Type ('$' being a shell prompt): 

         $ java com.gemplus.opencard.service.sim.Test
      or 

         $ java com.gemplus.opencard.service.sim.toolkit.Test

    2. And check the logs compared to the ones provided (i.e., respectively sim/test.log or sim/toolkit/test.log).

If during this test you encounter any error message, check the Troubleshooting Section to see if hints are available for this error.

Note that this code is currently in alpha version and so limited User's documentation is provided at this stage.

Please See the Javadocs for more details on the package API.

Appendix B. GLOSSARY

This appendix is informative.

 

 APDU

An APDU, Application Protocol Data Unit, is the basic command unit for a smart card. An APDU contains either a command message or a response message, sent from the card reader to the smart card or from the card to the reader.

 

 API

An API, Application Programming Interface, is an abstract definition of a service that can be provided either by a server, a component, an object class, etc. In the Java world, an API can be a Java Interface or a set of interfaces and abstract or concrete classes. For servers or other computing languages, it can be a set or functions (such as the C-based PKCS#11 API) or a language independent API defined in an IDL, such as the OMG IDL.

 

 OCF, OpenCard Framework

OCF, the OpenCard Framework, is a standardized, easy-to-use framework for implementing Smartcard-enabled solutions and Smartcard-based services. OpenCard Framework capitalizes on the broad, cross-platform benefits of Java, providing an open architecture and a set of common APIs (Application Program Interfaces) geared for this purpose. For more details about the basic architecture, concepts, and objectives of the OpenCard Framework, see the General Information Web Document and other documentation.
